# 家庭英语学习记录小程序设计

日期：2026-07-16

状态：待用户最终审阅

目标平台：微信小程序

技术路线：uni-app + uniCloud 阿里云免费服务空间

## 1. 产品概述

本产品是一个仅供单个家庭使用的英语学习记录小程序。它帮助家庭成员共同管理一个孩子的多门英语课程，记录剩余课时、固定课表和临时改期，在课前提醒指定成员，并在课后沉淀短视频、图片、PDF、网盘链接和学习笔记。

产品的核心价值是把“还剩几次课”和“每节课学到了什么”放在同一个家庭工具中，同时保持界面和操作足够简单。

## 2. 目标与非目标

### 2.1 第一版目标

- 一个家庭空间只管理一个孩子。
- 一个孩子可以有多门英语课程。
- 多位家庭成员共享数据，并支持多个管理员。
- 每门课程分别设置提醒成员。
- 支持每周固定课表和单次改期、请假、取消。
- 管理员课后手动确认完成，系统只扣除一次课时。
- 课后可立即添加学习记录，也可以跳过后稍后补充。
- 按时间线浏览成长记录，并按课程筛选。
- 使用 uniCloud 免费额度运行，不产生固定月费。
- 支持数据导出，避免数据被平台锁定。

### 2.2 第一版不做

- 多孩子切换。
- 培训机构、班级、老师账号和多家庭管理。
- 收费、续费、工资、招生和营销功能。
- 在线直播课堂。
- 完整课堂视频的云端托管和在线播放。
- 自动识别课程内容、AI总结和单词测评。
- 安卓原生应用和其他小程序平台。

## 3. 用户与权限

### 3.1 角色

| 角色 | 数量 | 权限 |
| --- | --- | --- |
| 家庭创建者 | 唯一 | 所有日常权限；任命或移除管理员；转让家庭；删除家庭 |
| 管理员 | 多个 | 管理课程、排课、扣课、撤销、上传资料、编辑学习记录、邀请查看成员 |
| 查看成员 | 多个 | 查看课程、日历、剩余课时和成长记录；不能修改 |

只有家庭创建者可以任命、降级或移除管理员。管理员可以邀请查看成员，但不能扩大其他人的权限。

### 3.2 邀请与成员失效

- 创建者或管理员可生成家庭邀请入口。
- 被邀请人通过微信身份加入家庭。
- 每个邀请入口有效24小时，只允许一位新成员成功加入，成功使用后立即失效。
- 成员被移除后，服务端立即拒绝其后续访问。
- 客户端缓存不作为权限依据。

## 4. 信息架构与导航

底部导航固定为四项：

1. 首页：剩余课时、最近课程、快速完成和最近学习。
2. 日历：周日历、固定课表和单次调整。
3. 成长：学习记录时间线及课程筛选。
4. 家庭：成员角色、操作记录、数据导出和空间用量。

首页采用“课程优先”布局。最接近上课时间的课程占据主卡片，直接显示课程名称、剩余次数、下次上课时间、提醒成员和“本次已上课”按钮。其他课程以紧凑列表呈现，最近学习记录位于课程信息之后。

## 5. 核心流程

### 5.1 创建家庭

1. 用户首次进入小程序并完成微信身份登录。
2. 用户创建家庭并填写孩子小名。
3. 系统把该用户设为唯一家庭创建者。
4. 创建者添加第一门课程和初始课时。
5. 创建者可邀请其他家庭成员。

孩子资料只要求小名或昵称，不要求身份证、学校、地址等非必要信息。

### 5.2 添加课程

课程包含：

- 课程名称。
- 可选图标或颜色标记。
- 初始总课时和当前剩余课时。
- 每次正常消耗课时数，第一版固定为1次。
- 固定上课规则，例如每周六10:00。
- 提醒时间，可选择提前一天、提前一小时或自定义；自定义值限制在上课前10分钟至7天之间，并以10分钟为粒度。
- 本课程提醒成员列表。

### 5.3 固定课表与单次调整

- 每门课程可以配置一个每周固定规则。
- 系统根据规则生成可见时间范围内的课程实例。
- 管理员可以只调整某一次课程，不影响后续固定规则。
- 单次课程状态包括：待上课、已完成、已请假、已取消和已改期。
- 选择修改固定规则时，界面必须与“仅修改本次”明显区分。
- 改期后，后续提醒以新时间为准，并通知该课程选中的提醒成员。

### 5.4 完成课程与扣课

系统不会因为时间到达而自动扣课。管理员课后点击“本次已上课”，确认界面明确展示剩余课时变化，例如“8 → 7”。

确认后有两个操作：

- 完成并添加学习记录：扣课后进入学习记录表单。
- 仅扣除课时：完成课程后直接返回，之后可从成长页面补写记录。

扣课操作必须是事务性的：

1. 校验当前用户是创建者或管理员。
2. 校验课程实例仍为待上课或允许完成的状态。
3. 校验剩余课时大于0。
4. 把课程实例改为已完成。
5. 把剩余课时减1。
6. 写入不可删除的操作日志。

每次请求携带唯一幂等编号。多位管理员同时操作同一课程时，只允许第一个请求成功，后续请求返回“本次课程已经完成”。

### 5.5 撤销

- 创建者和管理员可以撤销错误扣课。
- 撤销通过相反事务把课时加回并恢复课程状态。
- 原操作日志不会删除，系统新增一条撤销日志并关联原操作。
- 如果课程数据在扣课后发生冲突修改，撤销应拒绝并提示管理员先查看当前状态。

### 5.6 学习记录

每条学习记录可以包含：

- 所属课程和课程实例。
- 本节主题。
- 学习笔记、老师反馈和孩子表现。
- 单词、句型或能力标签。
- 短视频、图片和PDF。
- 完整课堂录像的网盘链接及可选提取码。
- 创建人、创建时间和最后修改人。

成长页面按时间倒序展示记录，并支持按课程筛选。文字表单在手机本地自动保存草稿，避免上传失败时丢失。

## 6. 文件与免费额度策略

第一版以保持零固定月费为约束，使用 uniCloud 阿里云免费服务空间。

### 6.1 直接上传

- 短视频：单段不超过1分钟和30MB，上传前压缩。
- 图片：上传前压缩，并保留满足手机查看的清晰度。
- PDF：单个文件不超过10MB。

### 6.2 外部链接

完整课堂录像不上传到 uniCloud，只保存家庭已有网盘的分享链接、提取码和备注。小程序不承诺网盘链接永久有效，失效后由管理员更新。

### 6.3 配额保护

- 家庭页显示云存储用量。
- 达到80%时提醒创建者清理或导出。
- 达到95%时暂停新文件上传，但继续允许课程、课时和文字记录操作。
- 免费空间续期前7天、3天和1天提醒创建者。
- 不自动开启无上限按量计费。
- 免费服务政策变化时，优先保证数据导出和迁移能力。

## 7. 提醒设计

- 每门课程维护独立的提醒成员列表。
- 每位成员可以关闭自己不需要的课程提醒。
- 定时云函数每10分钟检查即将达到提醒时间的课程。
- 提醒任务按“课程实例 + 提醒时间 + 成员”生成唯一键，避免重复发送。
- 微信提醒依赖成员主动订阅授权；未授权、额度不足或发送失败时不影响课程数据。
- 发送结果写入提醒日志，避免无休止重试。
- 首页和日历始终是提醒失败时的产品内兜底。

## 8. 技术架构

### 8.1 客户端

- 使用 uni-app 开发并编译为微信小程序。
- 客户端负责页面展示、输入校验、草稿缓存、上传进度和登录态刷新。
- 客户端不得直接决定用户权限、课时余额或扣课结果。

### 8.2 服务端

- 使用 uniCloud 阿里云免费服务空间。
- 使用云对象或云函数实现身份、权限、课程操作、扣课事务、提醒、文件元数据和导出。
- 查询可以使用受数据库 Schema 权限约束的 clientDB；所有关键写入必须通过云对象或云函数。
- 微信密钥只保存在服务端，不写入小程序代码。

### 8.3 数据库

| 集合 | 主要职责 |
| --- | --- |
| `families` | 家庭、孩子昵称、创建者、免费空间状态 |
| `family_members` | 微信用户与家庭的关系、角色、状态 |
| `courses` | 课程、初始课时、剩余课时、提醒配置、版本号 |
| `course_reminder_members` | 课程与提醒成员的多对多关系 |
| `schedule_rules` | 每周固定规则、生效范围和时区 |
| `lessons` | 每一次课程的计划时间、实际时间、状态和规则来源 |
| `learning_records` | 主题、笔记、标签、网盘链接和课程关联 |
| `attachments` | 云存储文件、类型、大小、所有者和删除状态 |
| `reminder_jobs` | 提醒计划、唯一键、发送状态和失败原因 |
| `operation_logs` | 扣课、撤销、改期、删除和角色变更审计记录 |

所有业务记录包含家庭ID。服务端根据登录身份查询有效成员关系，任何来自客户端的家庭ID都不能直接作为授权依据。

### 8.4 课程实例生成

- 固定规则不一次性生成无限未来课程。
- 日历查询时生成或补齐有限时间窗口内的课程实例。
- 每个课程实例由课程、规则和原计划日期形成稳定唯一标识。
- 单次改期修改课程实例，不修改固定规则。
- 修改未来固定规则时，从用户选择的生效日期开始使用新规则。

## 9. 异常处理

- 扣课、撤销、改期和角色变更必须等待服务端成功后再更新最终界面。
- 网络失败时保留用户输入并提供重试，不推测操作已经成功。
- 客户端超时后再次查询操作结果，使用幂等编号判断是否已成功。
- 文件逐个上传，单个附件失败不阻塞其他附件。
- 学习记录可先保存文字，再继续补传附件。
- 微信提醒失败只记录状态，不回滚课程或改期。
- 云服务不可用时显示只读缓存和明确的离线提示，不允许离线扣课。
- 数据冲突返回当前服务端版本，引导用户刷新后再次操作。

## 10. 隐私与数据删除

- 未成年人资料按敏感信息处理，只收集实现功能所必需的数据。
- 文件默认仅有效家庭成员可访问，不使用永久公开链接。
- 服务端签发10分钟有效的文件访问地址，并在每次申请时校验家庭成员关系。
- 普通学习记录删除后进入30天回收站，再异步删除关联文件。
- 回收站恢复必须同时恢复记录和仍存在的附件引用。
- 删除整个家庭仅限创建者，并要求再次输入孩子昵称确认。
- 家庭删除进入30天延迟删除流程；期间只有原创建者可以申请恢复，到期后再彻底删除。
- 被移除成员不能继续获得新的文件访问地址。

## 11. 数据导出与备份

- 创建者可以导出课程、课表、课程实例、剩余课时、成员、学习记录和操作日志。
- 结构化数据导出为JSON，同时提供便于阅读的CSV清单。
- 文件导出提供附件清单和分批下载，不生成超大单一压缩包。
- 网盘链接以原字符串导出。
- 导出文件只允许创建者在短时间内下载，到期后自动删除。

## 12. 测试策略

### 12.1 单元测试

- 角色权限矩阵。
- 固定课程生成、跨月和时区计算。
- 单次改期、请假、取消和固定规则变更。
- 扣课和撤销的状态机。
- 幂等请求和提醒唯一键。
- 配额阈值和上传限制。

### 12.2 集成测试

- 两个管理员并发完成同一课程，只扣除一次。
- 事务失败不会留下“课程已完成但课时未减少”等半完成状态。
- 被移除成员立即失去数据库和文件权限。
- 只向课程选择的成员创建提醒任务。
- 上传中断后可以继续保存文字并重试附件。
- 回收站恢复和到期删除行为正确。
- 导出数据完整且不包含其他家庭数据。

### 12.3 真机验收

- 至少两部不同品牌安卓手机。
- 多个微信账号分别扮演创建者、管理员和查看成员。
- 弱网、断网、重复点击和小程序切后台场景。
- 微信订阅授权允许、拒绝和失效场景。
- 免费额度达到80%和95%的界面与限制。

## 13. 第一版验收标准

- 创建者能在10分钟内完成家庭、课程和固定课表初始化。
- 受邀成员加入后能在一分钟内看到相同课程数据。
- 管理员能在三次点击内完成一节课并扣除一次课时。
- 并发或重复点击不会重复扣课。
- 单次改期不影响后续固定课程。
- 每门课程能分别选择提醒成员。
- 完成课程后能立即或稍后添加学习记录。
- 成长记录可以按时间浏览和按课程筛选。
- 查看成员无法执行任何写操作。
- 配额保护、导出、回收站和权限失效符合本设计。

## 14. 实施边界

实施计划应把项目拆成可独立验证的模块：项目骨架与登录、家庭与权限、课程与固定课表、课程实例与扣课事务、提醒、成长记录与文件、配额与导出、整体测试和发布准备。每个模块完成后都应有自动化测试或可复现的真机验收步骤。
